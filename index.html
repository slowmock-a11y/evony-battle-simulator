About

The Evony Battle Simulator models march-vs-march PvP combat using community-researched battle mechanics. Configure two armies, set buff percentages, and watch the engagement unfold step-by-step on a spatial battlefield.

Key Features

  • Army Configuration — Set troop counts per type and tier (T1–T14) for attacker and defender, with ATK/DEF/HP buff percentages.
  • Presets & Mirroring — Save army configurations as presets; mirror attacker to defender for quick symmetric setups.
  • Step / Round / Full Playback — Advance the battle one attack at a time, one full round, or run to completion, with adjustable speed.
  • Spatial Battlefield — A 5,200-unit linear field with troop movement, range indicators, and attack arrows.
  • Battle Log & Summary — Filterable log of every attack event, plus a side-by-side loss comparison panel.

Not Yet Modeled

These real-game mechanics are excluded from the current simulator:

  • Generals (base stats, ascending stars)
  • General Skills & Specialties
  • Equipment on generals
  • Research / Technology buffs
  • Civilization & Alliance buffs
  • Monarch Gear & Sub-city buffs
  • Wounded / Killed ratio
  • Wall / Trap mechanics (city attacks only)
  • Rally mechanics (multi-march)

Troops attack sequentially by range (longest range fires first):

Within each phase: attacker strikes first, then defender's surviving troops counter. Higher tiers attack before lower tiers.

Battlefield & Movement

Linear battlefield, 5,200 units long. Attacker starts at position 0, defender at 5,200.

Attacker (0) ————— 5,200 units ————— Defender (5200)

Each round, before attacking, each troop type evaluates movement:

  • Check: Is any enemy in range from current position?
  • Yes → Hold. Stay and fire at highest-priority target in range.
  • No → Advance. Move forward by speed units. Fire if now in range.

Speed = distance per round. Troops never reverse. When two opposing troops would overlap, they stop at melee range (50 units apart).

Damage Formula

damage = troopCount × ATK × modifier × ATK / (ATK + DEF)
kills  = floor(damage / target_HP)
  • troopCount — surviving count of the attacking layer
  • ATK — per-troop attack stat of the attacker
  • DEF — per-troop defense stat of the target
  • modifier — type matchup multiplier (see below)
  • target_HP — per-troop HP of the target

The ATK/(ATK+DEF) ratio ensures damage is always > 0 and high DEF reduces damage proportionally.
